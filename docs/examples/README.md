## Как использовать примеры

Каждый пример содержит:

1. **Описание** - Что это за приложение и зачем оно нужно
2. **Сущности** - Какие данные нужны
3. **API эндпоинты** - Какие endpoints создать
4. **Пошаговая реализация** - Как добавить функциональность
5. **Примеры кода** - Готовый код для копирования

## Общий подход

Все примеры следуют одной структуре:

1. **Domain** - Определяем сущности
2. **Repository** - Создаем интерфейсы и реализации
3. **Use Case** - Реализуем бизнес-логику
4. **Handler** - Создаем HTTP endpoints
5. **Router** - Добавляем маршруты

## Быстрый старт

Выберите интересующий вас пример и следуйте инструкциям. Каждый пример независим и может быть реализован отдельно.
